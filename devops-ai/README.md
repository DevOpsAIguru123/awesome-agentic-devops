# DevOps AI Client Bootstrap

Generated by `scripts/setup_devops_ai_clients.py`.

This folder contains a local, dependency-light MCP catalog server and DevOps operating skill for:

- Claude Code
- OpenAI Codex
- Cursor
- Google Antigravity
- VS Code

## One-command setup

From the repository root:

```bash
python3 scripts/setup_devops_ai_clients.py --clients all --write
```

The command writes project-local config only. It does not install third-party MCP servers, mutate global settings, or store secrets.

## What is configured

- `devops-ai/official-devops-mcp-catalog.json` — 48 official DevOps/SRE/Cloud MCP catalog entries extracted from `data/repos.yaml`.
- `devops-ai/DEVOPS_SKILL.md` — shared operating instructions for AI coding agents.
- `scripts/devops_catalog_mcp_server.py` — tiny stdio MCP server exposing catalog search tools.
- Client config files such as `.mcp.json`, `.cursor/mcp.json`, `.vscode/mcp.json`, `.codex/mcp.json`, and `.antigravity/mcp.json` depending on selected clients.

## Supported MCP tools

- `search_devops_mcp_catalog` — search by keyword/category.
- `get_devops_mcp_entry` — return one scored catalog entry by name.

## Production safety

This setup intentionally exposes the catalog, not your cloud accounts. To connect a real official MCP server, first review its catalog entry for:

- `action_level`
- `human_approval`
- `evidence_tracing`
- `risk_notes`
- credential scope and blast radius

Add real server credentials only through each client or platform's secure secret mechanism.
